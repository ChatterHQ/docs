The Partner API exposes a merchant's commerce, loyalty, and affiliate data. This page explains how the pieces fit together so the endpoints in the API Reference make sense.

## Merchant

Your API key belongs to a single **merchant** — the business you're integrating on behalf of. Every request is scoped to that merchant automatically; there's no merchant ID to pass. Whether calls read sandbox or live data is decided by the key's prefix (see [Authentication](/authentication)).

## Stores

A merchant operates one or more **stores** (storefronts). Most marketplace endpoints are scoped to a store — you'll frequently pass a `storeId` query parameter or path segment to say which storefront you mean.

## Conventions

- **Base URL** — every endpoint lives under `https://api.usechatter.app`, with a `/v1` prefix on paths.
- **JSON** — requests and responses use `application/json`.
- **Collections are paginated** — list endpoints return a `meta` + `results` envelope. See [Pagination](/concepts/pagination).
- **Errors use standard HTTP status codes** — see [Errors](/concepts/errors).

<Note>
  The Partner API is a strict subset of the full Hushscale platform — only routes

</Note>
