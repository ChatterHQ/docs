The **Chatter Partner API** lets external partners integrate with a merchant's Chatter account programmatically. It's a curated subset of the full Chatter platform — only the routes explicitly opened up for partner integrations are documented here.

Every request is authenticated with a merchant **API key** (`sk_live_…` or `sk_sandbox_…`). See [Authentication](/authentication) to get started.

<Card title="Authenticate your first request" icon="key" href="/authentication">
  Create an API key, choose between the `Authorization: Bearer` and `X-Api-Secret`
  header, and make a test call in a couple of minutes.
</Card>

## Base URL

All endpoints are served from a single production host:

```
https://api.usechatter.app
```

Use a `sk_sandbox_…` key to exercise the same endpoints against sandbox data without touching production balances or inventory.

## How the reference works

Open the **API Reference** tab in the top navigation. Every operation has:

- A full request/response schema generated from the live OpenAPI spec.
- A **Try it** playground — set your API key once and call the endpoint from the browser.
- Language code samples you can copy into your integration.

<Note>
  Playground calls hit **real** endpoints with your **real** key. Use a
  `sk_sandbox_…` key while you're testing so you don't affect production data or
  consume live quota.
</Note>
