This guide takes you from zero to a working API call. You'll create a key, authenticate, and list a merchant's stores.

## Prerequisites

- A Chatter merchant account with access to the [dashboard](https://dashboard.usechatter.app).
- A tool that can make HTTP requests (cURL, Postman, or your language's HTTP client).

## 1. Create an API key

<Steps>
  <Step title="Open the dashboard">
    Go to [dashboard.usechatter.app](https://dashboard.usechatter.app) and sign in.
  </Step>
  <Step title="Generate a key">
    Create an API key. Start with a **sandbox** key (`sk_sandbox_…`) so your first
    calls don't touch live data.
  </Step>
  <Step title="Copy it somewhere safe">
    You'll only see the full key once. Store it as an environment variable rather
    than pasting it into code:

    ```bash
    export CHATTER_API_KEY="sk_sandbox_your_key_here"
    ```
  </Step>
</Steps>

## 2. Make your first request

List the stores on the merchant account. Authenticate with the `X-Api-Key` header:

```bash
curl https://api.usechatter.app/v1/marketplace/stores \
  -H "X-Api-Key: $CHATTER_API_KEY"
```

A successful call returns a paginated JSON body:

```json
{
  "meta": { "count": 3, "currentPage": 1, "totalPages": 1, "next": null, "prev": null },
  "results": [
    { "id": "store_...", "name": "..." }
  ]
}
```

<Note>
  Whether a key hits **sandbox** or **live** data is determined by the key's
  prefix — the base URL is the same for both. See [Authentication](/authentication).
</Note>

## 3. Explore in the playground

Open the **API Reference** tab, pick any endpoint, click **Authorization**, and paste your key. The **Try it** button runs the request from your browser and shows the live response.
